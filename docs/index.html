▸ the self-improving loop

It doesn't just run. It gets better.

Most agents forget every run. They re-discover the same footguns forever. This one closes the loop: what one run pays to learn, the next run already knows.

Learn · /auto-learn

Harvests the run's checkpoint (blocked tasks, gate failures, thrash) and verifies each candidate lesson against the evidence (one flake is not a law). Survivors route to the layer that changes behavior; machine defects surface to you, never self-patched.

Map · /auto-map

Refreshes a disclosure-tiered context map: a lean root plus a per-folder CLAUDE.md that loads only when the agent works there. Every path, command, and claim is verified against the real repo before it ships.

Feed forward

Lessons land in the memory Claude Code loads automatically: CLAUDE.md, path-scoped .claude/rules, and auto memory. So the next run starts with them in context, no step required. Run N+1 starts with run N's tuition.

▸ the problem

Every agent can loop. The failure modes are what kill you.

Autonomy is easy. Autonomy you can leave running is not. These four are designed out, in code, not prose.

✗
Fakes the green.

"Fixes" the suite by skipping the red test. Every added test is mutation-checked; adding .skip to a test file is blocked at the tool layer.

✗
Grinds forever.

Runs three hours past the point of progress. Every loop declares a termination set (done-condition, cap, budget, no-progress stop) and stops the instant one fires.

✗
Poisons the history.

git add -As unrelated work into a commit, then force-pushes it. Bulk staging and plain force-push are blocked during a run; every task is its own clean rollback point.

✗
Lies about done.

Reports "done" for gates that never ran. Gates fail closed: a gate that did not execute is never reported clean, and a red validate is reported red.

▸ the autonomy contract

What separates autonomous from runaway.

Every skill honors the same five rules. It is the whole point of the collection.

Bounded, never infinite

Every loop declares a termination set and stops the instant one fires, and compiles onto native /goal + /loop.

Fails closed

A gate that did not run is never reported clean. A loop out of budget returns the open items. No fabricated green verdicts.

Verifies before acting

Findings that drive edits pass an adversarial panel: independent skeptics try to refute them. A dead panel fails closed, not open.

Durable and resumable

Long runs write a live status file and skip everything already done on resume, at any point, session-independent. Every call retries 10× through a rate-limit storm or a dropped connection; a longer outage just resumes from the checkpoint when the link returns.

Escalates, never guesses

Irreversible or ambiguous decisions stop and ask. Approval in one context does not extend to the next.

Measures, never assumes

An optimization is accepted only when a benchmark proves it faster with no correctness regression. Otherwise it is reverted.

▸ the collection

Eighteen skills. Four layers.

Phases run the lifecycle. Primitives are the machinery they compose. The autonomy layer chains them and makes them self-improving. Every one installs and runs on its own.

Phases· the lifecycle · one skill per step
/auto-spec

Turns a request into a grounded, testable spec. Recons the real repo, drafts objectives and acceptance criteria, then runs a completeness-critic loop until no gap or untestable claim remains. No invented requirements.

/auto-plan

Decomposes a spec into a DAG of atomic tasks, each with a disjoint write scope (≤3 files) and a slice-scoped validate. A deterministic validator proves the graph acyclic, topologically ordered, and race-free before any build.

/auto-build

Walks the DAG behind one approval: each task built test-first in an isolated worktree, integrated, slice-reviewed, fixed in a bounded loop, and committed on its own. Never builds on an unintegrated dependency; resumes from its checkpoint.

/auto-simplify

Loops until dry removing duplication, dead code, and over-abstraction, proving each edit behavior-preserving before keeping it. Respects Chesterton's Fence: never removes what it hasn't understood.

/auto-test

Finds untested behaviors, writes real tests, and loops until green. Every test is mutation-checked (break the code, the test must fail), so tautologies are rejected. De-flakes, and never games the suite green.

/auto-review

Reviews the diff across correctness, security, tests, and API dimensions at once, then keeps only findings that survive an adversarial skeptic panel. Dead panels fail closed. Optionally fixes the confirmed blockers.

/auto-performance

Baselines a benchmark, profiles to the real hotspots, and accepts an optimization only when a re-benchmark proves it faster with no correctness regression. Unmeasured "wins" are reverted.

/auto-ship

Runs fail-closed pre-launch gates (validate, review, security, rollback), assembles grounded release artifacts, and opens the PR or stages the rollout. The irreversible deploy stays human-gated.

Primitives· the machinery every phase composes
converge-loop

The bounded self-correcting loop behind every phase: act → measure → decide, in until-green or until-dry mode, with anti-thrash, no-progress, and budget stops. Compiles onto native /goal + /loop.

adversarial-verify

Proves a finding by trying to disprove it: N independent skeptics attack it through distinct lenses, and it survives only on a majority. Ties and dead panels fail closed: the gate that turns "plausible" into "verified".

checkpoint-resume

A durable, lock-serialized status file recording every unit's state, fully timestamped. Resume skips everything already done: session-independent, crash-safe, zero lost writes under concurrency. A read-only status reader renders any run live; refuses to clobber one.

fan-out-work

Covers a large work-list in parallel with concurrency caps and worktree isolation, aggregating faithfully. Never silently truncates: items discovered = covered + failed + dropped.

budget-guard

The five stop conditions every unattended run must declare (done-condition, hard cap, token/time budget, no-progress rule, escalation triggers), enforced so a run bounds itself or reports an honest partial.

Autonomy· chain · watch · schedule
autonomous-pipeline

Chains all eight phases end-to-end behind a single plan approval, with fail-closed gates between them, a durable checkpoint (resume at any phase or task), and a verified findings register at the end. One pass, never a runaway loop.

watch-and-act

Waits on an external signal the harness can't notify you about (CI, a deploy, a queue) on a cache-aware cadence, bounded by a deadline, and acts on the transition. Never polls work the harness already tracks.

schedule-recurring-agent

Stands up a recurring cron routine for standing work (triage, CVE watch, audits): a self-contained idempotent brief, a cadence matched to the work, a per-run budget, and a teardown condition.

Context & learning· the self-improving loop
auto-map

Builds a disclosure-tiered context map: a lean root CLAUDE.md, path-scoped rules, and a per-folder CLAUDE.md that loads only when the agent works there. A code gate verifies every path, command, and claim against the real repo.

auto-learn

Harvests each run's checkpoint into verified lessons and writes them where Claude Code auto-loads context: CLAUDE.md, path-scoped .claude/rules, or auto memory. The next run starts with them in context. Machine defects surface to you.
